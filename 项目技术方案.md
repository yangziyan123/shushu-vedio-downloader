# 鼠鼠视频下载工具技术方案

## 项目目标

构建一个轻量的视频解析与下载 Web 工具。用户输入视频链接后，后端通过 `yt-dlp` 解析视频信息和可用格式，前端展示格式列表，用户可以下载视频或提取 MP3 音频。

## 技术栈

- 前端：Vue 3、Vite、Axios、Lucide 图标
- 后端：FastAPI、yt-dlp
- 部署：Docker Compose、Nginx、Uvicorn

## 系统边界

当前版本只保留视频下载工具能力：

- `POST /api/parse`：解析视频链接
- `POST /api/download`：创建短期下载票据
- `GET /api/download`：根据票据或参数下载文件
- `POST /api/direct-url`：获取直连地址
- `GET /api/image-proxy`：代理封面图片
- `GET /api/health`：健康检查

明确移除并不再提供：

- AI 总结、字幕问答、思维导图和大模型接口
- Cookie 保存、导入、查询和清除
- 用户注册、登录、邮箱验证、密码重置和 JWT
- 免费额度、会员、订单、支付和人工开通

## 后端流程

1. 前端提交视频链接到 `/api/parse`。
2. 后端调用 `yt-dlp`，只解析元数据和格式，不下载文件。
3. 后端归一化格式列表，合并视频轨和最佳音频轨，并补充 MP3 音频选项。
4. 用户选择格式后，前端调用 `/api/download` 创建 5 分钟有效的下载票据。
5. 浏览器打开下载地址，后端在临时目录中下载文件，完成后通过 `FileResponse` 返回。
6. 后端通过后台任务清理临时下载目录。

## 数据与配置

当前没有数据库依赖。后端只需要下载临时目录：

- `DATA_DIR`
- `DOWNLOAD_DIR`
- `CORS_ORIGINS`
- `API_BASE_URL`

## 合规与限制

- 不承诺下载受 DRM、付费墙或平台权限保护的内容。
- 不保存用户平台登录态或 Cookie。
- 不提供绕过平台限制的能力。
- 下载能力取决于 `yt-dlp` 对目标平台的公开支持情况。
